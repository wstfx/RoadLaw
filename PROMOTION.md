## 推广节奏

### 第一天：可信首发

- 完成 GitHub Pages、README、Issue 表单和仓库 About 信息。
- 在朋友圈／同学群发布短文案，请真实使用者反馈移动端和内容问题。
- 不使用“包过”“保证通过”等承诺，强调免费、系统和可纠错。

### 第一周：展示具体价值

- 分别发布“记分矩阵”“交通标志易混组”“考前20分钟清单”三个内容切片。
- 每条内容只解决一个具体问题，并链接到对应页面。
- 根据 Issue 修复问题，在仓库 Release 或 README 记录更新。

### 后续：搜索与社区积累

- 每月核对法规版本和失效链接，更新 sitemap 的 `lastmod`。
- 为高频搜索问题补充独立、可分享的说明页时，再扩展 sitemap。
- 邀请驾考学习者提供真实可复现的问题；优先修复内容准确性和手机体验。
- 观察 GitHub Traffic 中的来源和热门内容，不购买虚假 Star 或刷量。
